## Project Overview

`ntistools` is a public R package for wrangling Nonprofit Trends and
Impacts Survey (NTIS) data, developed by the Urban Institute. It
provides pipe-friendly data-cleaning functions that replace repetitive
[`dplyr::case_when()`](https://dplyr.tidyverse.org/reference/case-and-replace-when.html)
/ [`dplyr::across()`](https://dplyr.tidyverse.org/reference/across.html)
patterns common in NTIS analysis scripts.

**This is a public repo. Never commit raw survey data, PII, internal
file paths, or credentials.**

## Common Commands

``` bash
# Install package locally (required before running tests)
R CMD INSTALL .

# Run all tests (package must be installed first)
Rscript -e 'library(testthat); library(ntistools); testthat::test_dir("tests/testthat")'

# Run a single test file
Rscript -e 'library(testthat); library(ntistools); testthat::test_file("tests/testthat/test-recode_binary.R")'

# Rebuild documentation (roxygen2 -> man/ and NAMESPACE)
Rscript -e 'roxygen2::roxygenise()'

# Full package check (if devtools is available)
Rscript -e 'devtools::check()'
```

## Architecture

Standard R package layout using `roxygen2` workflow:

- **R/**: Exported functions, one per file. Each takes a data frame as
  first argument and returns the modified data frame (pipe-friendly).
- **man/**: Auto-generated from roxygen2 comments. **Never edit
  directly.**
- **NAMESPACE**: Auto-generated by roxygen2. **Never edit directly.**
- **tests/testthat/**: One test file per function
  (`test-{function_name}.R`). Uses testthat edition 3.
- **vignettes/**: `before-and-after.Rmd` — side-by-side comparisons of
  raw dplyr vs ntistools.
- **inst/examples/**: Example NTIS cleaning scripts showing patterns
  these functions replace.
- \*\*\_pkgdown.yml\*\*: pkgdown site config. Site auto-deploys to
  gh-pages on push to main.
- **.github/workflows/**: GitHub Actions for pkgdown deployment.

## Key Conventions

- **Dependencies**: Only `dplyr` and `rlang`. Keep it minimal.
- **Column selection**: Most functions accept column names as character
  vectors via `vars` parameter (used with
  [`dplyr::all_of()`](https://tidyselect.r-lib.org/reference/all_of.html)).
  Exception: `combine_binary` and `count_binary` use `...` with unquoted
  tidyselect names.
- **Survey sentinel values**: NTIS uses numeric sentinels: 98 = “don’t
  know”, 97 = “not applicable”, 0 = “none”. Many function defaults
  reflect these.
- **NA propagation**: When all source columns are NA, derived columns
  should also be NA (not 0). This is deliberate across `combine_binary`,
  `count_binary`, and `apply_filter`.
- **roxygen2 with markdown**: Enabled via
  `Roxygen: list(markdown = TRUE)` in DESCRIPTION.
- **rlang imports**: `.data` and `:=` are imported from rlang in
  `R/ntistools-package.R`.

## Adding a New Function

1.  Create `R/{function_name}.R` with roxygen2 documentation and
    `@export` tag
2.  Create `tests/testthat/test-{function_name}.R` with testthat tests
3.  Run `Rscript -e 'roxygen2::roxygenise()'` to update NAMESPACE and
    man/
4.  Run `R CMD INSTALL .` then run tests to verify
5.  Add a before/after section to `vignettes/before-and-after.Rmd`
6.  Add the function to the appropriate group in `_pkgdown.yml`
