## Project Overview

`ntistools` is an R package for wrangling Nonprofit Trends and Impacts
Survey (NTIS) data, developed by the Urban Institute. It provides
reusable data-cleaning functions that replace repetitive
[`dplyr::case_when()`](https://dplyr.tidyverse.org/reference/case-and-replace-when.html)
/ [`dplyr::across()`](https://dplyr.tidyverse.org/reference/across.html)
patterns common in NTIS analysis scripts. Internal use only; contains no
raw survey data or PII.

## Common Commands

``` bash
# Run all tests
Rscript -e 'testthat::test_dir("tests/testthat")'

# Run a single test file
Rscript -e 'testthat::test_file("tests/testthat/test-recode_binary.R")'

# Check package (runs R CMD check)
Rscript -e 'devtools::check()'

# Rebuild documentation (roxygen2 -> man/ and NAMESPACE)
Rscript -e 'devtools::document()'

# Install package locally
Rscript -e 'devtools::install()'

# Build vignettes
Rscript -e 'devtools::build_vignettes()'
```

## Architecture

Standard R package layout using `devtools`/`roxygen2` workflow:

- **R/**: All exported functions, one per file. Each function takes a
  data frame as first argument and returns the modified data frame
  (pipe-friendly).
- **man/**: Auto-generated from roxygen2 comments in R/ files. Never
  edit directly.
- **NAMESPACE**: Auto-generated by roxygen2. Never edit directly.
- **tests/testthat/**: One test file per function
  (`test-{function_name}.R`). Uses testthat edition 3.
- **vignettes/**: `before-and-after.Rmd` shows how each function
  replaces raw dplyr code.
- **inst/examples/**: `sample_data_cleaning_script.R` is a real-world
  NTIS cleaning script showing the patterns these functions replace.

## Key Conventions

- **Dependencies**: Only `dplyr` and `rlang`. Keep it minimal.
- **Column selection**: Functions accept column names as character
  vectors via `vars` parameter (used with
  [`dplyr::all_of()`](https://tidyselect.r-lib.org/reference/all_of.html)).
  Exception: `combine_binary` and `count_binary` use `...` with unquoted
  (tidyselect) names.
- **Survey sentinel values**: NTIS uses numeric sentinels — 98 = “don’t
  know”, 97 = “not applicable”, 0 = “none”. Many functions have defaults
  reflecting these conventions.
- **NA propagation pattern**: When all source columns are NA, derived
  columns should also be NA (not 0). This is a deliberate design choice
  used across `combine_binary`, `count_binary`, and `apply_filter`.
- **roxygen2 with markdown**: Enabled via
  `Roxygen: list(markdown = TRUE)` in DESCRIPTION. Use markdown syntax
  in roxygen comments.
- **rlang imports**: `.data` and `:=` are imported from rlang in
  `R/ntistools-package.R` for tidy evaluation.
